获取Git仓库

要使用Git对我们的代码进行版本控制，首先需要获得Git仓库。

获取git仓库的通常两种方式：

工作区、暂存区、版本库慨念

版本库：前面看到的.git隐藏文件夹就是版本库，版本库中储层了很多配置信息、日志、版本信息等

工作区：包含.git文件夹的目录就是工作区，也称为工作目录。主要用于存放开发的代码

暂存区：.git文件夹中有很多文件，其中一个index文件就是暂存区，也可以叫做stage。暂存区是一个临时保存修改文件的地方

Git工作区中文件的状态

Git工作区中的文件存在两种状态：

  • untracked 未跟踪（未被纳入版本控制）

  • tracked 已跟踪 （被纳入版本控制）

    1、 Unmodified 未修改状态

    2、 Modified 已修改状态

    3、 Staged 已暂存状态

注意：这些文件的状态会跟随我们执行Git的命令发生变化
